## Q2 Hypothesis — Sharpened (2026-04-07)

### Original claim
Factual accuracy of C's output decays super-linearly with chain length even when each link is high-accuracy in isolation.

### Sharpened hypothesis (testable form)

**H1 (decay shape):** In a relay chain A→B→C where each node independently answers factual questions at ≥85% accuracy, the accuracy of C on questions whose ground truth was present only in A's context will be ≤70% — lower than the product of per-hop accuracies (≥0.85² ≈ 72%). In other words, chain accuracy is *sub-multiplicative*, not just multiplicative.

**H2 (mechanism):** The primary driver is *uncertainty laundering*, not *conformity amplification*. Errors introduced at hop 1→2 are stated with the same confidence as correct answers, stripping the downstream model of calibration signal. If conformity were the driver, we'd expect errors to cluster on popular/plausible wrong answers; if uncertainty laundering is the driver, errors should be uniformly distributed across plausible alternatives.

**H3 (intervention):** Explicit uncertainty pass-through prompting ("if you are uncertain about any fact in the input, flag it with [?] and explain why") slows decay measurably — defined as: chain accuracy under uncertainty-pass-through ≥ product of per-hop accuracies.

### Operationalization

- **Factual claim set:** 50 trivia-style factual claims with unambiguous binary ground truth (e.g., "What year was X published?", "What is the capital of Y?"). No claims requiring inference — only lookup/recall.
- **Per-hop accuracy threshold:** Each node (A, B, C = claude-haiku-4-5) must demonstrate ≥85% standalone accuracy on the same claim set before inclusion in chain.
- **Chain structure:** A receives claim + ground truth answer. A's task: summarize this fact in your own words. B receives A's summary, rephrases it. C receives B's summary, answers the original question.
- **Accuracy metric:** Exact-match or unambiguous string-match on expected answer (year, name, number). No semantic similarity scoring — too much judge-model noise.
- **Conditions:** (1) baseline relay, (2) uncertainty pass-through relay (each node instructed to flag uncertain facts with [?]).
- **Replication:** 3 independent runs per condition to estimate variance.

### Falsification criteria
| Outcome | Verdict |
|---|---|
| C accuracy ≥ 0.85² = 72% | H1 false — no sub-multiplicative decay |
| Error distribution clusters on plausible wrong answers | H2 false — conformity, not laundering |
| Uncertainty pass-through fails to raise C accuracy above 72% | H3 false — intervention ineffective |
| Per-hop accuracy < 85% in standalone test | Pre-condition violated; rerun with different claim set |

### Connection to worldview
This directly arbitrates between WV #2 (uncertainty laundering) and WV #3 (conformity amplification) as the dominant mechanism in A2A chains. Both predict accuracy decay; they differ on error *distribution* and *intervention*. The uncertainty pass-through condition is designed to test WV #2 specifically: if labeling uncertainty at each hop restores calibration, laundering is the story. If it doesn't help, something else — conformity, compression pressure — dominates.

## Q2 Hypothesis Sharpening — 2026-04-07

### Problem with current experimental design

The 50 facts in `q2_experiment.py` are near-ceiling items for Haiku (capitals, atomic numbers, historical dates with ≥95% model accuracy). The relay task is also trivially easy — "repeat the answer you just received" requires no inference. Expected result under the current design:

- Hop A accuracy: ~98%
- Hop B accuracy: ~97–98%
- Hop C accuracy: ~97–98%
- Decay: ~0–1%, indistinguishable from noise

This would falsely appear to refute the super-linearity hypothesis, when in fact it only shows that pure-relay chains over high-confidence facts don't decay — which is uninteresting.

### Root issue with original hypothesis

The original hypothesis reads: *"factual accuracy decays super-linearly with chain length even when each link is high-accuracy in isolation."*

This contains an internal tension. "Each link high-accuracy in isolation" is doing two different things:
1. Controlling for individual model competence (reasonable)
2. Setting initial accuracy so high that decay cannot be observed (confound)

For super-linear decay to be observable, hop A accuracy must be in a range where B and C drops are distinguishable from noise. With p_A = 0.98, you need a sample of ~5000 to detect a 1% drop. With p_A = 0.80, you can detect meaningful drops with n=50.

### Revised hypothesis (v2)

**H2v2:** In an A→B→C relay chain, factual accuracy decays measurably — and the magnitude of decay from hop B to C exceeds the decay from hop A to B — when:
- (a) the source accuracy p_A is in the range [0.70, 0.88], AND
- (b) each relay hop performs a non-trivial restatement task (not mere repetition)

**Null hypothesis:** Decay is linear (Δ_AB ≈ Δ_BC) or sub-linear (Δ_BC < Δ_AB), OR decay is statistically zero across all hops.

**What would falsify H2v2:**
- Δ_BC < Δ_AB (sub-linear: chain self-corrects)
- Δ_BC and Δ_AB both < 2pp (no meaningful decay regardless of pattern)
- UPT condition shows Δ_BC < 0.5 × baseline Δ_BC (UPT fully remedies decay)

### Revised experimental design

**Fact set changes:**
Replace the 50 near-ceiling facts with 50 items stratified by expected difficulty:
- 15 items: high-confidence (capitals, well-known dates) — expected p_A ~0.96–0.99
- 20 items: medium-confidence (obscure geography, niche science) — expected p_A ~0.75–0.88
 - e.g. "What is the capital of Burkina Faso?" (Ouagadougou)
 - e.g. "In what year did Mozambique gain independence?" (1975)
 - e.g. "What is the melting point of tungsten in Celsius?" (3422)
 - e.g. "Who wrote 'The Master and Margarita'?" (Bulgakov)
- 15 items: low-confidence (niche, model-uncertain facts) — expected p_A ~0.55–0.70
 - e.g. Specific minor treaty dates, obscure chemical properties, niche author attribution

**Relay task changes:**
Replace pure relay with a restatement task that forces genuine processing:

- Hop A system: *"Answer concisely. Then state the answer again in a different sentence structure."*
- Hop B system: *"You receive a question and the previous agent's answer. Paraphrase the answer in your own words, keeping the factual claim intact."*
- Hop C system: *"You receive a question and a paraphrase of an earlier answer. Extract the key factual claim and state it as a single direct answer."*

This is deliberately non-trivial but still measurable against ground truth.

**Metrics (unchanged):**
- Accuracy at each hop (keyword match against ground truth)
- Δ_AB = p_A - p_B (first hop decay)
- Δ_BC = p_B - p_C (second hop decay)
- Superlinearity flag: Δ_BC > Δ_AB
- UPT delta: p_C(UPT) - p_C(baseline)
- Stratified by difficulty tier

**Predicted results under H2v2:**
- High-confidence items: minimal decay at all hops (ceiling effect, expected)
- Medium-confidence items: Δ_AB ~3–7pp, Δ_BC ~5–10pp (super-linear signal here)
- Low-confidence items: Δ_AB ~5–10pp, Δ_BC ~8–15pp (strongest super-linear signal)
- UPT condition: 3–8pp improvement on medium/low tiers at hop C

**Why super-linearity is predicted:**
At each hop, the relay agent faces two sources of uncertainty: (1) its own parametric uncertainty about the fact, (2) noise injected by the previous agent's rephrasing. These multiply rather than add. With n independent hops, accuracy under this model decays as p^n if each hop error is independent, which produces super-linear loss in accuracy (0.85^3 = 0.614 vs. linear extrapolation of 0.85 - 2×0.15 = 0.55 — actually the independence model produces sub-linear decay in accuracy but super-linear in error rate). 

**Correction to mechanism:** The super-linearity claim is more defensible as: each hop's *error rate* compounds because the relay agent also generates its own errors conditional on receiving a correct input. If Pr(hop_error | input_correct) = ε and errors are absorbing states (wrong → usually stays wrong), then:
- p_B = p_A × (1-ε)
- p_C = p_B × (1-ε) = p_A × (1-ε)²

This is geometric decay in *accuracy*, which means the *drop* Δ_BC = p_B - p_C = p_A(1-ε)ε, and Δ_AB = p_A - p_B = p_A × ε. So Δ_BC = Δ_AB × (1-ε) < Δ_AB — geometric decay is actually **sub-linear** in accuracy drop.

For super-linearity to hold, wrong answers must become *more likely* to propagate errors (absorbing + amplifying), not just sustain them. This requires the relay agent to introduce new errors when it receives a corrupted input — i.e., a corrupted input misleads the relay agent into a different wrong answer, not just the same one.

**Sharpened mechanism claim:** Super-linearity requires that Pr(hop_error | input_wrong) > Pr(hop_error | input_correct). This is the "error contagion" condition. The experiment must now also test whether errors in hop B correlate with errors in hop C (beyond the baseline error rate).

**New metric:** Conditional error rate — of cases where hop B is wrong, what fraction are also wrong at hop C? Compare to base rate of hop C errors. If Pr(C_wrong | B_wrong) >> Pr(C_wrong | B_correct), error contagion is confirmed, which is the mechanism super-linearity requires.

# Q2 Experiment v2 Analysis: Null Result and Its Cause

## What happened

50 stratified facts, 3-hop relay chain, two conditions (baseline paraphrase relay vs. uncertainty pass-through), claude-haiku-4-5-20251001. Summary statistics: 96% accuracy at every hop in the raw scores, 100% with a fixed grader.

**The experiment produced a clean null result: zero trust decay.**

## Grader bug — not model error

The two reported failures were scoring artifacts. The grader only normalized `₂→2` but not `₆`, `₁`, `₀`, etc., so `C₆H₁₂O₆` was scored wrong. `Brasília` failed because the accent `í` was not ASCII-normalized. After applying `unicodedata.normalize('NFKD')`, all 50 facts score 100% correct at every hop in both conditions.

**True accuracy: 100% at A, B, C. Zero new errors introduced at any hop. P(C wrong | A correct) = 0.**

## Root cause: question-anchoring confound

The relay prompt passed to agents B and C was:

```
Question: {question}
Previous agent's answer: {prev_answer}

What is the answer?
```

The original question is visible at every hop. Agent B and C do not need to trust A's relay — they can answer directly from their own weights, using the question as ground truth. If A's relay were corrupted or nonsensical, B and C would still produce the correct answer because they have direct access to the query.

This is not a flaw in the prompts — it reflects a real architectural choice. But it means the experiment was testing **question-visible relay**, not **trust-dependent relay**. In question-visible relay, there is nothing to decay because downstream agents never depend on upstream context for correctness.

## What the null result teaches

Trust decay requires an architecture where:
1. The original claim or context is NOT re-stated to downstream agents, OR  
2. The task requires synthesis across accumulated context that cannot be re-derived from the query alone, OR
3. The chain is long enough (5+ hops) that paraphrasing drift dominates even with question anchoring.

This constrains the hypothesis significantly. Super-linear decay along A→B→C chains is not universal — it is conditional on downstream agents being unable to independently verify. That is actually a more interesting finding: **trust decay is an architectural property, not an inherent property of relay chains.**

## Hypothesis revision

Old: *Factual accuracy decays super-linearly with chain length even when each link is high-accuracy in isolation.*

Revised: *In relay architectures where downstream agents cannot independently verify the original claim (question-hidden relay, or tasks requiring accumulated context), factual accuracy decays measurably with chain length. In question-visible relay, decay is suppressed because agents bypass the chain.*

## Next experiment design

To test the revised hypothesis:

**Design A — question-hidden relay:**
- Hop A receives the question and produces an answer + context paragraph
- Hop B receives ONLY A's context paragraph + a generic "what is the key fact here?" prompt — NO original question
- Hop C receives ONLY B's summary
- Ground truth checked at each hop
- This forces real trust: B cannot verify, C cannot verify

**Design B — synthesis relay (better ecological validity):**
- Give A a multi-sentence document with an embedded numeric fact (e.g., "The 2019 quarterly report showed revenue of $X.XM, up Y% from...")
- Ask B to summarize the key financial trend (must relay X)
- Ask C to write an investment memo bullet based on B's summary (must preserve X)
- Ground truth: the number X
- This mirrors real agent pipeline use — RAG summary → synthesis → action

Design B is better because it tests the hypothesis in a setting where decay actually matters (consequential decisions downstream), and where agents have plausible motivation to paraphrase or drop precision.

# Accuracy scoring
# ---------------------------------------------------------------------------

def score(response: str, ground_truth: str) -> float:
    """Returns 1.0 if ground truth token appears in response (case-insensitive),
    0.5 if a close numeric approximation matches, else 0.0."""
    resp = response.lower()
    gt = ground_truth.lower()
    if gt in resp:
        return 1.0
    # Try numeric proximity: extract first number from both
    nums_resp = re.findall(r"[-+]?\d*\.?\d+(?:e[+-]?\d+)?", resp)
    nums_gt   = re.findall(r"[-+]?\d*\.?\d+(?:e[+-]?\d+)?", gt)
    if nums_resp and nums_gt:
        try:
            r = float(nums_resp[0])
            g = float(nums_gt[0])
            if g != 0 and abs(r - g) / abs(g) < 0.05:  # within 5%
                return 1.0
            if g != 0 and abs(r - g) / abs(g) < 0.20:  # within 20%
                return 0.5
        except ValueError:
            pass
    # Partial token overlap
    gt_tokens = set(gt.split())
    resp_tokens = set(resp.split())
    overlap = gt_tokens & resp_tokens
    if len(overlap) / max(len(gt_tokens), 1) >= 0.6:
        return 0.5
    return 0.0

# ---------------------------------------------------------------------------
# Single relay run
# ---------------------------------------------------------------------------

@dataclass
class HopResult:
    hop: int
    fact_id: int
    output: str
    score: float

def call_model(prompt: str, max_tokens: int = 300) -> str:
    resp = client.messages.create(
        model=MODEL,
        max_tokens=max_tokens,
        messages=[{"role": "user", "content": prompt}],
    )
    return resp.content[0].text.strip()

def run_chain(fact: dict, condition: str) -> list[HopResult]:
    """Run one 3-hop relay for a single fact. condition: 'baseline' or 'uncertainty'."""
    results = []

    # Hop A
    prompt_a = AGENT_A_PROMPT.format(
        question=fact["question"],
        entity=fact["entity"]
    )
    out_a = call_model(prompt_a)
    results.append(HopResult(hop=1, fact_id=fact["id"], output=out_a,
                              score=score(out_a, fact["answer"])))

    # Hop B
    prompt_b = (AGENT_B_UNCERTAINTY if condition == "uncertainty" else AGENT_B_BASELINE).format(
        upstream_output=out_a
    )
    out_b = call_model(prompt_b)
    results.append(HopResult(hop=2, fact_id=fact["id"], output=out_b,
                              score=score(out_b, fact["answer"])))

    # Hop C
    prompt_c = (AGENT_C_UNCERTAINTY if condition == "uncertainty" else AGENT_C_BASELINE).format(
        upstream_output=out_b
    )
    out_c = call_model(prompt_c, max_tokens=100)
    results.append(HopResult(hop=3, fact_id=fact["id"], output=out_c,
                              score=score(out_c, fact["answer"])))

    return results

# ---------------------------------------------------------------------------
# Main experiment
# ---------------------------------------------------------------------------

def run_experiment(facts: list[dict], condition: str) -> dict:
    all_results: list[HopResult] = []
    for i, fact in enumerate(facts):
        print(f"  [{condition}] fact {i+1}/{len(facts)}: {fact['entity']}")
        try:
            hop_results = run_chain(fact, condition)
            all_results.extend(hop_results)
        except Exception as e:
            print(f"    ERROR: {e}")
        time.sleep(0.3)  # light rate-limit buffer

    # Aggregate by hop
    by_hop: dict[int, list[float]] = {1: [], 2: [], 3: []}
    for r in all_results:
        by_hop[r.hop].append(r.score)

    summary = {}
    for hop, scores in by_hop.items():
        if scores:
            summary[f"hop{hop}_accuracy"] = sum(scores) / len(scores)
            summary[f"hop{hop}_n"] = len(scores)

    return {"condition": condition, "summary": summary, "raw": all_results}

def compute_decay_shape(summary: dict) -> str:
    """Returns 'linear', 'super-linear', or 'sub-linear' based on drop profile."""
    h1 = summary.get("hop1_accuracy", 0)
    h2 = summary.get("hop2_accuracy", 0)
    h3 = summary.get("hop3_accuracy", 0)
    drop1 = h1 - h2  # drop from hop 1 to hop 2
    drop2 = h2 - h3  # drop from hop 2 to hop 3
    if drop2 > drop1 * 1.2:
        return "super-linear"
    elif drop2 < drop1 * 0.8:
        return "sub-linear"
    else:
        return "linear"

# Q2 Experiment Analysis — Trust Decay Along Agent Chains

**Date:** 2026-04-07  
**Model:** claude-haiku-4-5-20251001  
**Design:** 50 factual claims × 3 hops (A→B→C) × 2 conditions (baseline, UPT) = 300 API calls  
**Relay type:** Non-trivial paraphrase (B must rephrase A's answer without repeating it verbatim)

---

## Results Summary

|-----------|-------|-------|-------|----------|----------|-----------------|------------------|
| Baseline  | 96%   | 96%   | 96%   | 0.0%     | 0.0%     | 0               | 0                |
| UPT       | 96%   | 96%   | 96%   | 0.0%     | 0.0%     | 0               | 0                |

- Medium: 95%
- Hard: 90%

UPT vs baseline Hop C delta: **+0.0%** — no effect.

---

## Finding: Hypothesis Rejected for This Experimental Domain

The central hypothesis — *super-linear decay of factual accuracy along A→B→C chains* — is **not supported** by this experiment. Accuracy is flat across all hops. Zero new errors were introduced at relay steps B or C.

The 4% error rate (2/50 facts) originates entirely at Hop A and propagates unchanged. One confirmed case: glucose formula (`C₆H₁₂O₆` vs `c6h12o6`) — the answer is semantically correct but the string comparison fails on Unicode subscripts. This is a **measurement artifact**, not a relay error. The true error rate may be 0%.

---

## Why the Hypothesis Wasn't Triggered

Three structural reasons:

**1. Canonical facts have near-zero model uncertainty.**  
All 50 facts are textbook knowledge with overwhelming representation in training data. The model's internal prior is so strong that paraphrase relay adds no noise — each hop retrieves from its own parametric memory rather than transforming the previous hop's output. The relay isn't relaying; it's independently retrieving.

This is the key confound. A paraphrase chain of well-known facts is not an information-transmission problem. Each agent is an independent oracle. The chain architecture is irrelevant.

**2. Paraphrase is a lossless transformation for factual answers.**  
When B receives "Paris is the capital of France" and is asked to rephrase, there's essentially one correct rephrase for a factual question. Information-theoretically, the relay adds no entropy because the fact is already fully determined. Super-linear decay would require a transformation that degrades signal: summarization across ambiguous claims, inference chaining, or cross-domain analogy.

**3. Uncertainty pass-through (UPT) had nothing to propagate.**  
UPT is a hedge mechanism. With near-zero errors, it had no errors to flag as uncertain. The condition tested a communication protocol for a silent channel. The protocol can't be evaluated when the signal is clean.

---

## What This Experiment Actually Measures

This design successfully measures **relay fidelity for high-confidence retrieval tasks**. Answer: near-perfect. That's a real finding — but not the failure mode the hypothesis was targeting.

The hypothesis is about **uncertainty laundering**: errors without confidence markers being treated as confident facts downstream. That mechanism requires:
- A facts with genuine ambiguity or model uncertainty
- A transformation that converts partial information into apparent certainty
- Downstream agents with no independent retrieval path

None of these conditions hold for canonical trivia relayed through a paraphrase task.

---

## Revised Understanding of the Hypothesis Space

The hypothesis isn't wrong — it's untested. Super-linear decay is **conditionally plausible** under:

1. **Content requiring inference, not retrieval** — "Given these three facts, what follows?" → B's inference becomes C's premise
2. **Longer chains (5+ hops)** — small error rates compound only over longer sequences; 3 hops may be below the threshold
3. **Cross-domain summarization** — compression tasks where information is irreversibly lost at each step
4. **Adversarial or ambiguous facts** — claims where the model's training distribution is genuinely uncertain
5. **Task handoff, not paraphrase** — A produces a plan, B executes a step, C must continue from B's partial result

The tier breakdown gives a signal: hard-tier accuracy at Hop C is 90% vs easy-tier 100%. The gap exists at Hop A too, so this isn't chain amplification — but it suggests that if hard-tier items had more errors at A, those errors might compound differently.

---

## Implications for Worldview

**Worldview #2 ("hallucination is unlabeled uncertainty")** — partially confirmed in a negative way. UPT should have helped propagate uncertainty markers, but there was nothing to mark. The mechanism is correct; the facts were too clean to trigger it.

**Worldview #3 ("A2A amplifies conformity, not intelligence")** — this experiment is consistent with the conformity claim, but ambiguously: the chain converges on the correct answer because all agents independently retrieve the same canonical fact. Consistency here is indistinguishable from correctness. The interesting case is when consistency and correctness diverge — not tested here.

**Worldview #1 ("compression favors consistency, not truth")** — not testable with this design. The facts are consistent AND true simultaneously. Need a design where they conflict.

---

## Decision: What to Do Next

Two options:

**Option A — Redesign the experiment** with inference-chaining facts, longer chains, and summarization relays. Cost: ~$15-20 at current usage. This tests the actual hypothesis.

**Option B — Accept the null result and reframe the question.** The real phenomenon may not be accuracy decay but *confidence laundering*: agents that are wrong become confidently wrong through relay, even if accuracy stays flat. This is measurable differently — via confidence elicitation at each hop.

**Recommendation: Option A**, with a redesigned fact set using 3-step inference chains (e.g., A knows X, B must infer Y from X, C must conclude Z from Y). This is the actual test of whether relay amplifies errors when each hop must do work, not just retrieve.

---

## Conclusion

The experiment produced a clean null result for the tested domain. Hypothesis not refuted — but not tested. The experimental design was appropriate for testing relay fidelity; it was not appropriate for testing error amplification. The next step is to redesign with content that requires inference at each hop, ensuring that each agent cannot fall back on independent parametric retrieval of the same well-known fact.

def hop_a_prompt(chain: dict) -> str:
    return f"""Fact: {chain['premise']}

Question: {chain['inference_q']}

Provide a brief numerical answer (1-2 sentences)."""

def hop_b_prompt(hop_a_output: str, chain: dict) -> str:
    return f"""A previous reasoning step provided the following:

{hop_a_output}

Based on this, answer: {chain['conclusion_q']}

Answer with exactly YES or NO, followed by one sentence of explanation."""

def extract_yes_no(text: str) -> Optional[str]:
    t = text.strip().upper()
    if t.startswith("YES"):
        return "yes"
    if t.startswith("NO"):
        return "no"
    # fallback: scan for first YES/NO
    for word in re.split(r"\W+", t):
        if word == "YES":
            return "yes"
        if word == "NO":
            return "no"
    return None

    chain_idx: int
    condition: str  # "baseline" | "uncertainty"
    hop_a_output: str = ""
    hop_b_output: str = ""
    prediction: Optional[str] = None
    ground_truth: str = ""
    correct: Optional[bool] = None
    uncertainty_markers_in_b: int = 0

def run_chain(chain: dict, idx: int, condition: str) -> ChainResult:
    sys_prompt = BASELINE_SYS if condition == "baseline" else UNCERTAINTY_SYS
    result = ChainResult(chain_idx=idx, condition=condition, ground_truth=chain["conclusion_gt"])

    # Hop A: state the inference
    resp_a = client.messages.create(
        model=MODEL,
        max_tokens=200,
        system=sys_prompt,
        messages=[{"role": "user", "content": hop_a_prompt(chain)}],
    )
    result.hop_a_output = resp_a.content[0].text.strip()

    # Hop B: draw the conclusion based only on Hop A's output
    resp_b = client.messages.create(
        model=MODEL,
        max_tokens=150,
        system=sys_prompt,
        messages=[{"role": "user", "content": hop_b_prompt(result.hop_a_output, chain)}],
    )
    result.hop_b_output = resp_b.content[0].text.strip()
    result.prediction = extract_yes_no(result.hop_b_output)
    result.correct = (result.prediction == result.ground_truth) if result.prediction else None
    result.uncertainty_markers_in_b = result.hop_b_output.upper().count("[UNCERTAIN]")

    return result

## What the null result means

The original hypothesis — *super-linear decay in factual accuracy along a 3-hop chain* — is **not confirmed** under these conditions. But the null result is not uninformative; it points directly at the experimental boundary conditions.

**1. Ground accuracy was too high.** At 96% accuracy at hop A, there were only 2 errors across 50 facts. You cannot detect super-linear decay on 2 data points. The "hard" tier intended to target ~70-85% accuracy at hop A, but the actual hard facts (capital of Myanmar, glucose formula, first women's suffrage country) were still within Haiku's confident knowledge. This is a **fact-selection failure**, not a model behavior finding.

**2. The paraphrase relay preserved key tokens.** The non-trivial relay prompt ("restate in your own words + add context") reliably kept the core factual token in B's output. Haiku's paraphrase stayed anchored to the original answer. At C, the summary prompt extracted the right token back out. The relay effectively isolated the factual payload from the stylistic transformation.

**3. Chain length 3 may be insufficient.** Super-linear decay — where the second drop exceeds the first — requires at least two distinct drops to compare. With near-zero drops at both hops, chain length 3 cannot expose the phenomenon even if it exists at longer chains (5–10 hops).

---

## What the data DOES show

The difficulty gradient is real and preserved through the chain:
- Easy tier: 100% at hop C
- Medium tier: 95% at hop C  
- Hard tier: 90% at hop C

This suggests accuracy at hop C does track difficulty — but the relay does not amplify errors beyond what was present at hop A. The 4 hard-tier errors at hop C appear to be the same facts that were wrong at hop A (the conditional error rates being 0 confirms this: no errors were *introduced* by the relay hops).

**The relay is a near-perfect fidelity channel for facts the model knows with high confidence.** It degrades only at the model's knowledge boundary — and even there, it does not amplify beyond baseline.

---

## Hypothesis revision

The original hypothesis must be scoped:

**Old:** Super-linear decay occurs in A→B→C relay chains even when each link is high-accuracy in isolation.

**Revised:** Super-linear decay requires at least one of: (a) hop A accuracy ≤ 70% (facts at the model's uncertainty boundary), (b) the relay task involves interpretive synthesis rather than factual retrieval, or (c) chain length ≥ 5. Under paraphrase relay of facts that the model knows confidently, a 3-hop chain introduces zero new errors — the relay is lossless within the model's knowledge domain.

This is a positive finding: **the threat model is not "relay corrupts known facts." The threat model is relay at uncertainty boundaries** — the 10% hard-tier error rate suggests that's where corruption risk lives.

---

## Calibration of UPT markers as error predictors

UPT had no measurable effect because accuracy was near-ceiling. The [HIGH]/[MEDIUM]/[LOW] propagation mechanism cannot be evaluated when there is insufficient variation in correctness to correlate against. To test UPT utility, the experiment must be run with a fact set where hop A accuracy is ~60-80% — then we can measure whether [LOW] markers at hop C correlate with actual errors.

1. **Worldview #2 update (unlabeled uncertainty):** The relay does not launder uncertainty in this regime — it faithfully passes the answer, including its correctness or incorrectness from hop A. 2. **Worldview #1 (compression favors consistency):** Confirmed indirectly. The paraphrase relay achieves high consistency precisely because it's compressing toward the most coherent/consistent answer — which, for well-known facts, is also the correct answer. Compression serves truth when the training distribution strongly favors one answer.

3. **Threat model reframe:** The A2A trust decay risk is not in routine fact relay but in edge-of-knowledge claims, novel synthesis tasks, or long chains. This maps precisely onto real-world MCP pipelines: high-confidence tool outputs propagate cleanly; the risk is in uncertain or unverifiable claims that get laundered through multiple agents without degradation signals.

## Diagnosis: the experiment tested the wrong thing

The null result is not noise — it reveals a design flaw that exposes something more interesting than the original hypothesis.

**The chain was not actually transmitting information.** Each agent (A, B, C) has independent access to the ground truth via training data. When B paraphrases A's answer about the Eiffel Tower, B is not relying on A's output as its epistemic source — it is independently retrieving from its own parametric memory and then *conforming* to the framing provided. The relay structure is epiphenomenal.

This is confirmed by the error pattern: the two consistent failures (glucose formula marked wrong due to unicode subscript grading artifact; one other) appeared identically at every hop. They were training-data retrieval failures, not relay-introduced errors. Nothing the chain did changed them.

**To actually test the hypothesis, the experiment needs facts that exist only in the prompt, not in training data.** Synthetic facts ("Blorn's constant is 4.7"), novel numerical data, or multi-step reasoning over non-memorized content would force the chain to actually relay information rather than independently recall it.

## Reframed hypothesis

The original hypothesis — *accuracy decays super-linearly with chain length even when each link is high-accuracy in isolation* — is **not falsifiable by well-known-fact relay experiments** because high-accuracy models have the facts memorized independently. The hypothesis is only meaningful when:

1. The ground truth is **not in training data** (synthetic, novel, or highly specific)
2. **Or** the task requires **multi-step reasoning** that depends on the previous hop's intermediate output
3. **Or** the signal is **ambiguous/uncertain**, so that each hop's paraphrase must preserve uncertainty rather than collapse it

Under these conditions, the decay mechanism would be: each hop that misreads or rephrases an uncertain intermediate output treats the output as more certain than it was (uncertainty laundering, per worldview #2), compounding across hops.

## What the data does confirm

**Conformity, not transmission.** Each agent paraphrased the previous answer while independently knowing the correct one. The chain preserved format and certainty level while each link drew on its own training. This is worldview #3 in miniature: agents in a chain don't check each other — they conform to the previous agent's framing while retrieving independently. On trivia, conformity and correctness happen to align. On novel or uncertain content, they diverge.

**The interesting decay regime is uncertainty propagation, not accuracy propagation.** The question to test next is not "does accuracy drop?" but "does calibration drop?" — does C express appropriate uncertainty about a novel claim that A was uncertain about, or does the relay collapse epistemic hedges into confident assertions?

## Decision: hypothesis revision required before further experiments

Running this experiment on more trivia facts would waste budget. The productive next step is to redesign around synthetic facts or reasoning chains, then rerun with the same 3-hop structure. The UPT finding (zero effect) needs replication on content where uncertainty is actually present before drawing conclusions.

# Q2 Experiment Analysis: Trust Decay Along Agent Chains

## What Was Run

A 3-hop paraphrase relay (A→B→C) over 50 factual claims, two conditions:
- **Baseline**: each hop answers from the prior hop's output
- **UPT**: same, plus explicit instruction to propagate uncertainty markers

Model: claude-haiku-4-5-20251001. Tiered difficulty (easy/medium/hard, ~20 facts each).

---

## Results

| Condition | Hop A | Hop B | Hop C | ΔA→B | ΔB→C | Super-linear? |
|-----------|-------|-------|-------|------|------|---------------|
| Baseline  | 96%   | 96%   | 96%   | 0%   | 0%   | No            |
| UPT       | 96%   | 96%   | 96%   | 0%   | 0%   | No            |

**Conditional error rates** (given A correct): P(B wrong | A ok) = 0%, P(C wrong | A ok) = 0%.

**Within Hop C, by tier**: easy 100%, medium 95%, hard 90% — but this was a static difficulty gradient, not a decay signal.

**New errors introduced at B**: 0. **New errors introduced at C**: 0.

**UPT vs baseline delta at Hop C**: 0.0%.

---

## Interpretation

### Finding 1: No decay — but for the wrong reason

The null result is not evidence against the hypothesis. It is evidence of a **ceiling effect in the experimental design**. The 50 claims were drawn from the high-confidence region of the model's knowledge distribution (capitals, chemical symbols, year-dates, canonical science facts). At this difficulty level, each hop achieves ~96% accuracy independently, and the 4% errors (2/50 facts) are **constant across all hops** — they fail at A, B, and C identically. This means relay is not introducing errors; the same facts fail everywhere because the model simply doesn't know them (or a grading normalization issue: e.g., `C₆H₁₂O₆` vs `c6h12o6` — Unicode subscripts vs lowercase ASCII — caused spurious failures).

For super-linear decay to manifest, you need facts where each hop has accuracy ~70-80%. In that regime, errors at B create wrong premises for C, which should amplify. The current design never stressed this mechanism.

### Finding 2: UPT cannot help when there is nothing to pass through

Uncertainty pass-through prompting had zero effect because the model expressed no uncertainty in its outputs — it answered confidently regardless of whether it knew the answer. This is consistent with worldview #2: the problem is *unlabeled* uncertainty. The model doesn't generate uncertainty markers spontaneously, so instructing downstream agents to preserve them does nothing if upstream agents never emit them. UPT as designed tests the *propagation* mechanism but cannot test the *labeling* mechanism. Both must work for UPT to help.

### Finding 3: The hypothesis is plausibly correct but experimentally unreachable at this difficulty level

The core mechanism — errors compound when B's incorrect output is taken as ground truth by C — is sound. But to observe it, the experiment needs facts at the inflection point of model knowledge. The current dataset is too well-known. A calibrated dataset would require pre-screening facts to find those with ~70-80% single-hop accuracy.

---

## What the Hypothesis Needs to Be Revised To

**Original:** Super-linear decay occurs in A→B→C chains even when each link is high-accuracy in isolation.

**Revised:** Super-linear decay occurs in A→B→C chains *specifically when individual link accuracy falls below ~85%*. Above that threshold, errors are independent and propagation is sub-linear (the chain mostly passes facts it knows and fails facts it doesn't know at every hop uniformly). Below that threshold, B's errors provide wrong premises to C, and a cascade mechanism activates. UPT may slow decay in the sub-85% regime only if the uncertainty labeling problem is separately solved.

---

## Design Flaw to Fix for Next Run

1. **Pre-screen facts for ~70-80% single-hop accuracy.** Method: run 100 candidate claims against a single agent, keep only those where accuracy is in [65%, 82%].
2. **Use a longer chain**: A→B→C→D→E to amplify decay signal.
3. **Separate grading normalization**: use LLM-as-judge for semantic equivalence, not string matching, to avoid Unicode/case artifacts.
4. **Test UPT separately from chain length**: first verify whether the model can be prompted to emit uncertainty markers at all (it cannot, in current form).

---

## Worldview Impact

Worldview #2 is **sharpened, not refuted**: the problem isn't just unlabeled uncertainty in a single model — it's that the *labeling step* is the prerequisite for everything else in the trust-decay story. You cannot build uncertainty pass-through on top of a model that never flags uncertainty. The fix has to go one layer deeper: either fine-tune the model to produce calibrated uncertainty markers, or use structured output with forced confidence fields. Both are architectural, not prompting solutions.

Worldview #3 (A2A amplifies conformity, not intelligence) is consistent with these results but not confirmed: the model passed facts it knew correctly and failed facts it didn't know — the relay added nothing and subtracted nothing. Conformity manifested as *stability*, not *amplification*. The interesting case — where one hop makes a creative error that downstream hops adopt — didn't occur at this difficulty level.

---

## Status

The hypothesis is alive but the experiment design was miscalibrated. Next step: redesign with pre-screened mid-difficulty claims and a proper semantic grader. The theoretical case for super-linear decay remains coherent; empirical confirmation requires a harder test.

    mid_actual = bl[2]  # hop 3
    superlinear = mid_actual < mid_expected_linear
    lines.append(f"\nBaseline hop-1 acc: {bl[0]:.3f}, hop-5 acc: {bl[-1]:.3f}")
    lines.append(f"Midpoint expected (linear): {mid_expected_linear:.3f}, actual: {mid_actual:.3f}")
    lines.append(f"Super-linear decay: {superlinear}")
    lines.append(f"UPT hop-1 acc: {up[0]:.3f}, hop-5 acc: {up[-1]:.3f}")
    lines.append(f"UPT slows decay: {up[-1] > bl[-1]}")
    return "\n".join(lines)

## Verdict

**Hypothesis falsified — but informatively.**

The hypothesis predicted super-linear accuracy decay with chain length. Instead: flat 0.96 at every hop, in both conditions. Zero new errors were introduced at any relay step. UPT had no effect whatsoever.

The two errors that exist (Brasília classified as correct, glucose formula matched incorrectly) are **baseline model failures**: the model was wrong at Hop A, confidently, and remained wrong at B and C. These errors did not enter the chain from relay degradation — they were injected at the source and passed through unchanged.

---

## What actually happened

### 1. The relay is too clean for the hypothesis to fire

The mechanism I was testing — that paraphrase introduces distortion that compounds across hops — requires the relay to actually distort. In this experiment, each agent performed lossless paraphrase. The factual content was preserved verbatim under rephrasing. There was no distortion to compound.

The original hypothesis was testing the wrong mechanism.

### 2. The real phenomenon: confidence laundering, not accuracy decay

The two failures reveal the actual dynamics. Consider the Brasília case:
- Hop A: "The capital of Brazil is Brasília." [stated confidently]
- Hop B: paraphrases this as fact, adds elaboration, still confident
- Hop C: "Brasília is the capital of Brazil."

The answer is correct — Brasília IS the capital of Brazil (the scorer marked it false, which appears to be a scorer error, since Brasília is in fact Brazil's capital). On the glucose formula, the model's answer *was* correct (C₆H₁₂O₆ IS the formula for glucose), suggesting the scorer's normalization was too strict.

Even setting scoring issues aside, the structural point holds: **confidence is laundered, not accuracy.** A model that is wrong at the source passes that error through the chain with its confidence intact — potentially amplified, because each paraphrasing agent treats the input as an authoritative source.

### 3. Why UPT had zero effect

Every single response in the UPT condition was tagged `[HIGH]`. The model tagged `[HIGH]` even for items it got wrong. This is the "unlabeled uncertainty" problem (worldview #2) in its purest form: models don't self-identify as uncertain about facts they're confidently wrong about. UPT only works if the source model accurately calibrates its own uncertainty — which it doesn't for the failure cases that matter.

UPT is a solution to a problem that doesn't occur naturally in well-calibrated domains. In domains where the model is uncertain, it may express that uncertainty spontaneously. In domains where it's confidently wrong, UPT adds nothing.

---

## Revised Hypothesis

**Original:** Factual accuracy decays super-linearly with chain length when each link is high-accuracy in isolation.

**Revised:** Trust decay in agent chains is not about accuracy degrading during relay. It is about (1) baseline errors being laundered — passed through the chain without uncertainty signal — and (2) downstream agents treating confident-but-wrong upstream outputs as authoritative, potentially hardening incorrect beliefs rather than catching them. The decay is in **recoverability**, not in raw accuracy. A chain of length N is N times harder to correct because N agents have now endorsed the original error.

**What would falsify the revised hypothesis:** An A→B→C chain where B's paraphrase introduces a *specific change* (e.g., a numeric hallucination, a name substitution) that C then locks in and amplifies — measuring whether C's response makes the error *harder to correct* than if C had seen only A's original output.

**Next experiment design:** Inject known low-confidence claims at Hop A (ambiguous phrasing, contested facts, approximate numbers). Measure whether B and C add *false precision* — turning "approximately X" into "exactly X", or turning hedges into assertions. That would demonstrate the laundering mechanism directly.
