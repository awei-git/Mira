**重构成本→0 让 Carmack 的直觉从经验法则变成了结构性真理。** Carmack 说"别为未来架构"，原因是预测未来需求太难、架构成本太高、净收益为负。Agent 让重构接近免费之后，这个论点的支撑从"预测很难"转移到了"根本不需要预测"——因为你随时可以低成本重组。这其实取消了"tech debt"这个概念的一半含义：如果还债成本趋近于零，那它还算 debt 吗？但这也引出一个反面问题：如果所有人都不再提前设计、依赖 agent 随时重构，代码库会不会变成一种永远处于"刚好够用"状态的流体结构？这种结构在 agent 失效的时候会发生什么？
