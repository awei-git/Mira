# Progressive Distillation

**Tags:** explorer, knowledge-management, reading, synthesis, notes

## Core Principle
Compress source material through multiple staged passes so that knowledge becomes more actionable with each pass. Resist distilling on first read — relevance reveals itself over time.

## The Five Layers

**Layer 1 — Capture**
Save the full source with minimal processing. Don't summarize yet. Tag by domain and source type only.

**Layer 2 — Bold**
On second encounter, bold the 10-20% of text that is genuinely important. Criteria: novel claim, evidence, or mechanism — not just interesting phrasing.

**Layer 3 — Extract**
From bolded passages, write 2-5 discrete, standalone propositions in your own words. Each must stand without surrounding context.

**Layer 4 — Implication**
For each extracted proposition: "This matters because ___ / This implies ___ for [relevant domain]." This is the layer that actually gets used.

**Layer 5 — Connect**
Link to existing knowledge: does this confirm, refute, or extend something already known? Update the knowledge structure accordingly.

## Quality Filter
Only 10-20% of captured items should reach Layer 4. The friction is the filter — if everything seems Layer-4 worthy, the filter is broken.

## Application
- First read of any article: Layer 1 only.
- Weekly review: promote most valuable captures to Layer 2-3.
- When writing a briefing: pull from Layer 3-4 items only.
