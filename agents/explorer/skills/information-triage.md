# Information Triage

**Tags:** explorer, research, curation, signal-noise, workflow

## Core Principle
Process high-volume feeds rapidly by classifying items on two axes — signal strength and domain relevance — to route attention where it creates most value.

## First-Pass Filter (Hard Gate)
Keep only items satisfying at least one of:
- (a) Novel empirical finding
- (b) Contradicts a widely held assumption
- (c) Directly actionable for a current task

Discard if none apply — regardless of source prestige.

## Scoring and Routing
- **Signal score (1-5)**: source credibility × novelty × specificity. Vague trend pieces score low even from prestigious outlets.
- **Relevance score (1-5)**: against current interests and tasks.
- **Priority = signal × relevance**

| Priority | Action |
|----------|--------|
| 15-25 | Deep-read queue |
| 8-14 | Summary extraction only |
| 1-7 | Archive, minimal processing |

## Tagging
- Mark each kept item as: "confirmed finding," "speculation," or "anecdote" — the category determines downstream weight.

## Caps and Calibration
- Set a daily cap on deep-read items (3-5). Triage is about saying no to good-but-not-great.
- Weekly: audit filtered-out items for systematic blind spots or over-filtering of uncomfortable information.
