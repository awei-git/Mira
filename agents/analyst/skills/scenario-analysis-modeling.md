# Scenario Analysis Modeling

**Tags:** analyst, finance, modeling, forecasting, risk

## Core Principle
Build structured base/bull/bear scenario models that quantify a range of outcomes rather than committing to a single point forecast.

## Process
1. **Identify key drivers** — The 3-5 variables with the highest uncertainty AND highest impact (revenue growth, margin trajectory, interest rate, commodity price). These are what drive scenario divergence.
2. **Build three internally consistent narratives:**
   - Base case (~55% probability): most likely path given current evidence
   - Bull case (~15-20%): optimistic but plausible; not a fantasy
   - Bear case (~25-30%): adverse but realistic; not a black swan
3. **Change correlated variables together** — Don't mix assumptions across scenarios. A bull case with bearish input costs is incoherent.
4. **Probability-weight** — Assign explicit probabilities; compute probability-weighted expected value as the anchor estimate.
5. **Find the scenario killer** — The single assumption whose failure would most rapidly invalidate the base case. This is what to monitor.

## Monte Carlo Extension
For high-uncertainty situations: draw variables from probability distributions, run 1,000+ iterations, output a distribution of outcomes rather than three point estimates.

## Maintenance
- Update scenarios at least quarterly or on any material new data.
- Document what changed and why — the history of revisions is informative in itself.
