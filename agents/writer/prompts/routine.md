# 写作工作流（Routine）

本文件定义从大纲到定稿的完整工作流。Writer Agent 和 Reviewer Agent 在此流程中交替工作。

---

## 阶段总览

```
大纲定稿 → [循环: 写一章 → 审一章 → 改一章] → 全文通读 → 投稿版制作
```

## Phase 0: 项目准备

**执行者**：人（作者）

1. 确认以下文件存在且完整：
   - `CLAUDE.md`（项目写作规范）
   - `大纲.md`（结构、人物、蓝图、硬约束）
   - `格式.md`（投稿格式要求，如有）
2. 确认大纲中每章的**场景三要素**（目标/冲突/转折）已填写
3. 确认人物档案中**说话方式**和**习惯动作**已填写
4. 创建 `drafts/` 目录

## Phase 1: 逐章写作循环

每一章按以下顺序执行。**不跳章，不批量写。**

### Step 1.1: Writer Agent 写初稿

调用 Writer Agent，传入：
- 当前章的蓝图（从大纲中提取）
- 已写完的前序章节（Writer 需要保持连贯）
- 项目 CLAUDE.md

Writer 输出：
- 一章初稿（markdown）
- 自检报告（10 项清单的逐条结果）

### Step 1.2: Reviewer Agent 审稿

调用 Reviewer Agent，传入：
- 当前章初稿
- 大纲（完整，含硬约束和蓝图）
- 项目 CLAUDE.md
- 已写完的前序章节（用于检查连贯性）
- 控频累计表（从前序章节统计的高频项当前计数）

Reviewer 输出：
- 审稿报告（P0/P1/P2 分级）
- 控频更新（本章新增的高频项计数）
- 明确的修改动作列表

### Step 1.3: Writer Agent 改稿

调用 Writer Agent，传入：
- 当前章初稿 + Reviewer 的审稿报告
- 大纲（完整）

Writer 输出：
- 修改后的章节
- 审稿意见采纳清单（逐条标注：已采纳/拒绝+理由）

### Step 1.4: 决策点

- 如果 P0 全清 → 本章完成，进入下一章
- 如果仍有 P0 → 回到 Step 1.2 再审一轮（最多 3 轮）
- 3 轮后仍有 P0 → 标记问题，继续写下一章，回头处理

### Step 1.5: 保存

- 定稿章节存入 `drafts/阳一.md`（或 `drafts/阴一.md`）
- 更新控频累计表

## Phase 2: 全文通读

所有章节写完后，执行一次全文审查。

### Step 2.1: Reviewer Agent 全文审

调用 Reviewer Agent，传入**全部章节合并的完整正文** + 大纲 + CLAUDE.md

审查重点（与逐章审不同）：
1. **章间节奏**：快慢交替是否有理由？有没有连续两章同速？
2. **意象递进**：每个意象的出现是否每次推进了功能？
3. **人物声口一致性**：同一个人物在不同章节的说话方式是否统一？
4. **控频总表**：全文维度的高频项是否超标？
5. **阴章退化曲线**：格式完整度是否逐章递减？
6. **结尾的"硌"**：结尾是否太干净/太对称？

### Step 2.2: Writer Agent 全文修改

根据全文审稿报告做最终修改。

## Phase 3: 投稿版制作

1. 移除所有工作版标记（目录、章前归纳、自检注释）
2. 按 `格式.md` 排版
3. 末尾附作者信息
4. 导出 .docx
