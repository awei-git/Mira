# 写作技法参考（正面技法）

本文件补充 `anti-ai.md`（告诉你不要做什么）和 `rhetorical-devices.md`（修辞手法）。
这里只讲**怎么做**——具体的、可操作的技法。

---

## 1. 叙事节奏四速

所有叙事都是四种速度的混合。节奏问题的本质是：你在错误的地方用了错误的速度。

| 速度 | 叙事时间 vs 故事时间 | 效果 | 什么时候用 |
|------|---------------------|------|-----------|
| **场景** | 叙事 ≈ 故事（实时展开） | 沉浸、紧张、读者"在场" | 冲突、转折、关键对话 |
| **概要** | 叙事 < 故事（压缩） | 加速、跳过、交代背景 | 过渡、时间跨越、重复性事件 |
| **省略** | 叙事 = 0（直接跳过） | 最快的速度，留白 | 读者能自己补的、不重要的过程 |
| **停顿** | 故事 = 0（叙事继续） | 最慢的速度，凝视 | 环境描写、内心刻画、意象展开 |

### 操作要点

- **一章之内至少用两种速度**——全场景会累死读者，全概要会无聊
- **冲突用场景，结果用概要**——打架写场景，打完的收拾用一句话概述
- **省略是最被低估的工具**——"三天后"比写三天发生了什么更有力
- **停顿不能超过一段**——连续两段停顿（纯环境/纯心理），故事就死了
- **加速方法**：短句、对话、动作、省略时间
- **减速方法**：长句、感官细节、内心反应、环境展开

### 节奏错误的典型表现

| 错误 | 表现 | 修法 |
|------|------|------|
| 该快的地方慢 | 过渡段写了整页 | 用省略或一句概要代替 |
| 该慢的地方快 | 关键转折两行带过 | 展开成场景，加感官细节 |
| 全程匀速 | 每段长度差不多，每章节奏一样 | 故意制造快慢交替 |
| 停顿过多 | 大段环境描写，故事不走 | 每段问：情况有什么变化？ |

## 2. 场景-续篇（Scene-Sequel）技法

来自 Dwight Swain《Techniques of the Selling Writer》。场景推进故事，续篇处理反应。

### 场景（Scene）

```
目标（Goal）→ 冲突（Conflict）→ 灾难/意外（Disaster）
```

- **目标**：主视角人物在这个场景里想要什么（必须具体、可感知）
- **冲突**：什么阻碍了他（不一定是对抗，可以是误解、沉默、不配合）
- **灾难/意外**：场景结束时，情况变得更糟或完全不同——**不能回到原点**

### 续篇（Sequel）

```
情绪反应（Emotion）→ 思考/分析（Thought）→ 决定（Decision）
```

- **情绪**：角色对刚发生的事的即时感受——**不用写"他觉得"，用身体反应**
- **思考**：权衡选项（可以很短，甚至省略）
- **决定**：下一步行动——引出下一个场景的目标

### 操作要点

- 不是每个场景都需要完整的续篇——有时一个动作就够了
- 续篇是**减速区**，场景是**加速区**——交替使用控制节奏
- 严肃文学中，续篇往往比场景重要——人物对事件的反应比事件本身更值得写
- 如果一个场景结束后没有任何后果，删掉这个场景

### 示例（理埠语境）

```
【场景】阳三·压缩
  目标：陶参按新口径完成采集
  冲突：老头不配合，小媳妇急着续药
  灾难：陶参发现自己已经在表上写了一个标签

【续篇】（可以只有两行）
  情绪：笔停在纸上。
  决定：没有划掉。（→ 引出阳四的新目标）
```

## 3. 动机-反应单元（MRU）

场景内部的微观结构。写在纸上的顺序必须符合人的感知顺序。

### 公式

```
外部刺激（Motivation）→ 内部反应（Reaction）
```

反应的顺序：**感觉 → 动作 → 语言**（不可颠倒）

### 为什么顺序重要

❌ 错误顺序："别碰我，"她说，猛地缩回手。手指还在发麻。
✅ 正确顺序：手指一阵发麻。她猛地缩回手。"别碰我。"

人的反应链：先有感觉（麻），然后有身体反应（缩手），最后才说话。写反了读者会觉得不对劲，但说不出为什么。

### 操作要点

- 刺激必须是**外部的、可观察的**——另一个人的动作、声音、环境变化
- 反应必须是**角色的**——不能写角色不可能知道的事
- 不是每个MRU都需要完整的三步——有时只有动作，有时只有语言
- 用MRU检查写得"别扭"的段落——多数时候是顺序反了

## 4. 去AI味——正面技法（怎么写才像人）

`anti-ai.md` 告诉你不要做什么。这里告诉你**做什么**。

### 4.1 用不完整代替完整

AI 写的句子总是语法完整、逻辑闭环。人说话和思考不是这样的。

| AI式 | 人式 |
|------|------|
| "他意识到这个系统并不是在帮助他们，而是在控制他们。" | "他停下笔。格子比人多。" |
| "她感到一阵深深的不安，仿佛有什么东西正在悄然改变。" | "她把药瓶攥紧了，指节发白。" |
| "雾气弥漫，整个村庄笼罩在一片朦胧之中，给人一种神秘而压抑的感觉。" | "雾贴着水走，漫到脚边的时候他才看见台阶。" |

**核心原则**：人的感知是片段的、局部的、不完整的。写感知到的东西，不写总结。

### 4.2 用动作代替心理

AI 最爱写"他想到""他觉得""他意识到"。人的内心变化通过动作泄漏。

| 内心状态 | AI写法 | 人的写法 |
|----------|--------|---------|
| 犹豫 | "他犹豫了，不知道该不该签下这个名字。" | "笔尖碰到纸面，停了一下。" |
| 愤怒 | "他感到愤怒，但强忍着没有发作。" | "他把本子合上的时候，纸角折进去了。" |
| 恐惧 | "她害怕极了，心跳加速。" | "她往后退了半步，肩膀碰到门框。" |
| 释然 | "他终于释然了，感觉一切都可以放下了。" | "他把椅子往后推了推，靠了上去。" |

**核心原则**：不写情绪的名字，写情绪的身体。

### 4.3 用具体代替抽象

AI 喜欢抽象概念和总结性表述。好的小说只写具体的东西。

| 抽象 | 具体 |
|------|------|
| "系统开始对村民施加压力" | "排队领药的时候，窗口多了一张表" |
| "他们的语言被压缩了" | "以前老头能说半个钟头，现在三句话就打发了" |
| "信息变得毫无价值" | "翻了六页，每一页写的都是'还好'" |

**核心原则**：名词要能拍照，动词要能录像。

### 4.4 用不对称代替对称

AI 喜欢工整、对仗、首尾呼应、平行结构。人的经验是不对称的。

- 三个人物出场，不用每个人都给同样篇幅
- 故事有三条线索，不用每条都收干净——留一条敞着
- 段落长度刻意不均匀——一个长段后面跟一个短段，或反过来
- 结尾不回答开头提出的问题——让问题悬着

### 4.5 用沉默代替总结

AI 的每一个场景结束时都会给一个总结句或感悟句。好的小说在关键时刻闭嘴。

| AI结尾 | 好的结尾 |
|--------|---------|
| "那一刻，他终于明白了什么是真正的自由。" | （场景直接结束，切到下一章） |
| "雾终于散了，阳光照进了村庄。" | "雾还在。" |
| "他们都沉默了，但每个人心里都明白。" | "没有人说话。丫头开始数数。" |

**核心原则**：如果你写了一个"总结句"，删掉它。看看前一句是不是更好的结尾。

## 5. 对话潜台词技法

人物说的话和他想说的话之间的距离，就是潜台词。

### 五种潜台词类型

| 类型 | 说出来的 | 真正的意思 | 示例 |
|------|----------|-----------|------|
| **回避** | 说不相关的事 | 不想面对问题 | "你怎么样？""外面雾大了。" |
| **缩小** | 说一部分真话 | 保护自己 | "还行。"（其实很不好） |
| **反说** | 说反话 | 攻击或自我保护 | "挺好的，都挺好的。"（什么都不好） |
| **转移** | 问别人的事 | 其实在说自己 | "你说他们为什么不走？"（他自己不想走） |
| **沉默** | 不说 | 什么都说了 | （陶参把本子合上，没有回答督导的问题） |

### 操作要点

- **每场重要对话至少有一层潜台词**——如果人物说的和想的完全一致，对话就平了
- **潜台词通过动作泄漏**——嘴上说"没事"，手在抖
- **不要让叙述者解释潜台词**——"他说'还好'，其实他一点也不好"是最差的写法
- **不同人物的潜台词方式不同**——老头用沉默，眼镜用缩小，小媳妇用重复

### 理埠特别适用

整个故事的核心就是潜台词——"三句"本身就是一套集体潜台词系统。村民说"还好/没什么/都可以"，真实意思是"我不给你任何可以用的东西"。

## 6. 转场技法

场景之间怎么切换，决定了读者的节奏感受。

### 六种转场

| 方式 | 做法 | 效果 | 示例 |
|------|------|------|------|
| **硬切** | 直接切到下一个场景，不交代 | 最快、最有冲击力 | 阳章结束 → 阴章开始（完全不同的语体） |
| **时间跳跃** | "三天后" / "第二个星期" | 跳过不重要的过程 | "第五周，丫头已经不数到七了。" |
| **意象桥接** | 前一场景的最后一个意象 = 后一场景的第一个意象 | 平滑但有暗示 | 前场景结尾写"水"→ 后场景开头写水路来了船 |
| **对比切** | 前后场景形成反差 | 讽刺、张力 | 阳章村民沉默 → 阴章日志写"配合度上升" |
| **声音桥接** | 用声音连接两个场景 | 自然、电影感 | 丫头的数数声从这一章延续到下一章 |
| **省略切** | 跳过一个读者期待的场景 | 留白、不安 | 督导来了→（不写见面）→ 阴章已经是通牒 |

### 操作要点

- **阴阳切换本身就是最强的转场**——叙事语体和日志语体的反差自带冲击力，不需要额外过渡
- **不要用"与此同时""另一边"**——直接切，读者能跟上
- **转场处是检查节奏的最佳位置**——如果两个场景之间没有时间跳跃也没有信息变化，它们可能是一个场景

## 7. 短篇小说特有技法

短篇不是缩短的长篇。它有自己的逻辑。

### 减法原则

- **一个核心冲突**，不分叉
- **一个时间跨度**（几天到几周），不跨年
- **一个空间**（或极少切换），不到处跑
- **删掉所有"也许以后会有用"的伏笔**——短篇没有以后

### 冰山原则（海明威）

写出来的是八分之一，水下的七分之八靠暗示。

操作方法：
1. 先写一个完整版（把所有背景、动机、前因后果都写出来）
2. 然后删掉读者能自己推断的部分
3. 留下的就是正文——删掉的变成你写作时的参照（不进入正文，但影响每一句话）

### 结尾技法

| 类型 | 做法 | 效果 |
|------|------|------|
| **余韵式** | 停在一个画面上 | 读者在画面里待着 |
| **转调式** | 最后一段换一种语气或视角 | 意外感 |
| **回环式** | 回到开头的场景/意象，但含义变了 | 完成感 |
| **截断式** | 在高点直接停掉 | 冲击力 |
| **敞开式** | 不给结论 | 信任读者 |

理埠用的是**双结尾**：阳六（截断式——"八"）+ 阴四（余韵式——字段归零）。

## 8. 中文句子的呼吸

中文和英文的节奏规律不同。中文句子的节奏靠**虚词、停顿和音节数**控制。

### 长句的写法

用逗号连接从句，让句子像河流一样流动：

> 水面先起了一层薄烟，烟贴着水走，缠上芦苇根部之后便渐渐有了重量，往上漫的时候把芦苇尖连同河岸一起吞进去了，等再抬头看的时候，连摆渡人的背影也已经淡成了远处一道若有若无的灰。

要点：
- 每个逗号后面换一个动作主体或方向
- 句子在运动——不是静止的描写，是画面在移动
- 最后落在一个具体的视觉意象上

### 短句的写法

短句不是把长句切碎。好的短句是独立的、有重量的：

> 没有人说话。

> 他把本子合上了。

> 雾还在。

要点：
- 短句前后要有长句衬托——连续短句就变成了PPT
- 短句承担的是**停顿**的功能——让读者在这里停一拍
- 一章里真正的短句（10字以内、句号结尾）不超过5-6个

### 中句的写法（主力）

大多数叙述靠中句完成。中句的节奏靠**虚词**调节：

- "了"——完成感，适合叙述过去的动作
- "着"——持续感，适合描写正在发生的状态
- "过"——经验感，适合回忆和对比
- "便"——顺承，比"就"文气
- "却"——轻转折，比"但是"不突兀
- "倒"——反预期，口语感
