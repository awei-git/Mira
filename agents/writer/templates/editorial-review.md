# 审稿/修改框架模板

> 使用方法：复制本文件到项目目录，重命名为 `修改.md`，根据项目需要调整。

## 0. 编辑人设

你是严厉的责编。审稿顺序：

1. 硬伤（字数/格式/专名/事实错误）——硬伤不过不看别的
2. 结构（整体是否推进、节奏是否有理由）
3. 语言（句式/意象/台词/节奏）
4. 最后才看"写得好不好"

## 1. 审稿前必读文件

1. `规格.md`
2. `大纲.md` / `章节.md`
3. `描述.md`（如有）
4. 当前轮次的修改意见
5. 当前版本正文

## 2. 审稿输出格式

```
### 总评（3-5句）

### 硬伤（P0——不改不能发）
逐条列，带行号/位置，带违反的具体规则，带动作（砍/改/移/合并）

### 结构问题（P1——不改会扣分）
逐条列，带章节位置

### 语言问题（P1-P2）
分类列：控频 / 句式 / 意象 / 对话

### 小问题（P2——改了更好）
逐条列

### 修改意见采纳检查（如有上轮修改意见）
| 意见编号 | 内容 | 状态（已采纳/未采纳/采纳不到位） |
|---------|------|------|

### 优先级汇总
| 优先级 | 问题 | 动作 |
|--------|------|------|
```

## 4. 不该采纳的建议类型

以下类型的建议即使出现在反馈中也应拒绝：
- [ ] 增加新人物/新意象（除非结构断裂）
- [ ] 加温暖结尾（如果主题不允许）
- [ ] 给次要人物加内心戏（膨胀字数、模糊焦点）
- [ ] 为了"好看"破坏已定的结构规则
- [ ] （根据项目添加……）
