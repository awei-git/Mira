# Incluing（隐式世界构建）

## One-liner
通过角色行为、假设和随口提及来传递世界观设定，而不是停下来解释 -- SF写作的核心信息传递技术。

## The Problem
SF需要让读者理解一个与现实不同的世界。最自然的冲动是解释："在这个世界里，人类已经发明了..." 这就是info-dump。Info-dump打断叙事节奏，暴露作者在场，把小说变成说明书。但如果不解释，读者又看不懂。Incluing是这个两难的解法。

## The Pattern

```
1. EMBED  — 把设定信息嵌入角色的日常行为和对话中。
             角色不会解释自己世界的规则，就像你不会解释重力。
2. IMPLY  — 用反应代替解释。角色对某事的反应暗示了
             这个世界的规则。
3. LAYER  — 每次只透露一层。读者在前3章拼出基本规则，
             到中段理解深层逻辑，结尾才看到全貌。
4. TRUST  — 信任读者的推理能力。留出空白让他们自己填。
```

## Techniques

**1. 假设即设定**
角色把世界规则当作理所当然。读者从"理所当然"中推导规则。
Bad:  "自从政府禁止了纸质书籍，人们只能在屏幕上阅读。"
Good: "她在黑市上摸到一本纸质书时，手指不知道该怎么翻页。"
（禁令、稀缺性、数字化常态 -- 全部隐含在一个动作里。）

**2. 行话与自然命名**
角色使用他们世界的行话，不加解释。读者从上下文推断含义。
Bad:  "他使用了一种叫做'跃迁'的超光速旅行技术。"
Good: "跃迁后他吐了三次，比上回好多了。"
（技术存在、有副作用、他经常使用 -- 三个事实，零解释。）

**3. 对比反应**
两个来自不同背景的角色对同一事物的不同反应，揭示了世界的规则。
"她第一次看到真正的树时哭了。他不明白为什么 -- 树到处都是。"
（她来自没有树的地方。他觉得正常。两个世界的差异通过情感反差传递。）

**4. 随口提及（passing reference）**
在对话或叙述中随口提到一个设定元素，不做任何解释，继续往前走。
"他绕过隔离区的边界 -- 那些信标已经三年没人维护了 -- 朝北走。"
（隔离区存在、正在衰败、至少三年了。作者不停下来解释为什么。）

**5. 逆向info-dump：角色不解释已知事物**
如果某件事在这个世界是常识，角色绝不会解释它。
Bad:  两个科学家互相解释他们都知道的基本原理。
Good: 两个科学家争论的是边缘案例和例外情况。读者从争论的内容反推基本原理。

## When to Use
- 开篇前三章（读者最容易被info-dump劝退的阶段）
- 引入新设定元素时（新技术、新地点、新物种）
- 角色进入一个对他们来说熟悉的环境时
- 任何你发现自己在"解释"的时候 -- 停下来改用incluing

## Common Pitfalls
- **过度隐晦**: 如果关键信息太隐蔽，读者不是"推理"而是"困惑"。核心规则可以在前几章给得稍微明确一些。
- **所有角色都知道一切**: 安排一个局外人角色（新手、外星人、穿越者）可以合理地让其他角色解释一些东西 -- 但不要滥用这个套路。
- **前后不一致**: Incluing要求你自己对世界规则有完整的理解。如果你自己没想清楚，隐含的信息会互相矛盾。
