# 小说/中篇写作框架

## 1. 项目启动

### 写之前必须回答的问题
1. **核心矛盾是什么？** 一句话说清楚（不超过30字）
2. **主角想要什么？被什么阻碍？** 欲望+阻力=故事
3. **结局是什么？** 不需要细节，但需要知道方向
4. **字数和发表目标？** 这决定了结构密度
5. **视角？** 第一人称/第三人称限制/全知/第二人称（各有代价）

### 项目文件搭建
```
项目名/
├── CLAUDE.md     # 继承 writings/CLAUDE.md，加入本项目特有的语言规范
├── 规格.md       # 字数、章节数、结构类型、投稿目标
├── 大纲.md       # 章节卡（见下方模板）
├── 描述.md       # 人物档案、世界观设定、意象系统
├── 修改.md       # 审稿方法、控频表、禁止操作
├── 格式.md       # 排版和投稿格式
├── drafts/       # 按版本存放草稿
└── submission/   # 定稿
```

## 2. 故事结构工具箱

根据项目需要选用一种或混合使用。

### 三幕结构（经典）
| 幕 | 占比 | 核心任务 |
|---|---|---|
| 第一幕：建置 | 25% | 日常世界→激励事件→踏入非常世界 |
| 第二幕：对抗 | 50% | 挑战升级→中点转折→一切塌掉 |
| 第三幕：解决 | 25% | 最终对决→高潮→新常态 |

### Save the Cat 15节拍（适合商业小说/类型小说）
1. 开场画面（日常世界的一帧）
2. 主题陈述（通常由配角无意中说出）
3. 铺垫（"六件需要修复的事"）
4. 催化事件（打破日常的事件）
5. 辩论（"我真的要做这件事吗？"）
6. 踏入第二幕（主动选择行动）
7. B故事（副线，通常是情感线）
8. 玩乐段落（"承诺的前提"兑现）
9. 中点（假胜利或假失败）
10. 反派逼近（压力升级）
11. 一无所有（最低谷）
12. 灵魂暗夜（内心最深的挣扎）
13. 踏入第三幕（新的理解/转变）
14. 终局（运用新能力解决问题）
15. 终场画面（与开场对照，展示变化）

### 起承转结（适合短篇、散文化小说）
| 段 | 功能 |
|---|---|
| 起 | 引入人物和处境 |
| 承 | 展开、深化，但不制造冲突 |
| 转 | 意外转折——不是情节反转，而是视角/理解的转换 |
| 结 | 余韵，不做总结，让读者自己体会 |

### 自定义结构
你可以设计自己的结构系统（如《无法访问》的阴阳双线结构）。关键是：
- 结构必须有**升级机制**——每一章/节比前一个推进了什么
- 结构必须有**节奏理由**——不是为了好看，而是为了叙事效果
- 结构必须能**自检**——可以用表格追踪每章是否空转

## 3. 场景写作

### 场景三要素
每个场景必须包含：
1. **目标**——场景中的主视角人物想要什么
2. **冲突**——什么阻碍了他
3. **转折**——场景结束时情况发生了什么变化（不能和开头一样）

### 场景节奏
- 进场晚，离场早——不要写人物走进房间、坐下、寒暄
- 行动→反应→决定→行动——推进的基本单位
- 一个场景1000-2000字——太短没展开，太长读者疲劳

### 场景检查
写完每个场景问自己：
- 删掉这个场景，故事还成立吗？→ 成立就删
- 这个场景的信息能合并到其他场景吗？→ 能就合并
- 读者在这个场景里知道了什么新信息？→ 没有就有问题

## 4. 人物写法

### 人物档案（最小可行版本）
- **名字**：全名+日常称呼
- **外在特征**：一个最突出的身体特征（不要面面俱到）
- **说话方式**：句子长度、口头禅、语气词、停顿习惯
- **核心欲望**：他最想要什么
- **核心恐惧**：他最怕什么
- **习惯动作**：2-3个标志性的肢体动作

### 人物纪律
- 遮住名字能分出谁在说话——如果不能，对话没写好
- 每个人物用不同的词汇量和句子结构
- 次要人物不需要内心戏——动作和对话足够
- 不增加计划外的有名字角色

### 对话写法
- 人物说话不是在做报告——会跑题、会停顿、会答非所问
- 潜台词比明说更有力——人物说的和想的不一样
- 对话标签："他说"就够了，动作标签代替说话标签更好
- 对话推进情节或揭示性格——不做这两件事的对话就删

## 5. 意象和象征

### 原则
- 意象必须从故事内部生长，不是从外部贴上去
- 每个意象每次出现必须推进——不能停在同一个功能上
- 全文跟踪意象出现位置和功能变化（用意象递进表）
- 隐喻只存在于结构中，正文绝不点破

### 意象递进表模板
| 意象 | 第1次 | 第2次 | 第3次 | ... |
|---|---|---|---|---|
| [具体事物] | [功能/含义] | [推进到] | [再推进到] | |

## 6. 改稿流程

### 第一轮：结构审查
- 对照大纲，检查每章是否完成了承诺的任务
- 找出空转章（没有新信息、没有推进的章节）
- 检查节奏：紧张-缓和的交替是否有理由

### 第二轮：语言审查
- 执行触觉动作控频（如果有控频表）
- 找碎片化断句、机械排比、格言堆叠
- 检查形容词密度
- 检查意象是否每次出现都有新功能

### 第三轮：对话审查
- 遮住名字测试
- 找"说破寓言"的台词——删掉这句话读者能自己得出结论吗？
- 检查对话标签是否过度花哨

### 第四轮：细节审查
- 人名、地名、时间线一致性
- 字数是否符合规格
- 格式是否符合投稿要求

## 7. 投稿准备

- 按照 `格式.md` 排版
- 准备投稿信（简介+内容梗概+作者简介）
- 记录投稿平台、日期、状态
- 收到退稿意见后存入 `审稿意见_r[N].md`，进入下一轮修改
