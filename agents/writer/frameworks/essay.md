# 个人散文 / Substack / 随笔写作框架

## 1. 散文的核心

散文不是"随便写"——散文是用个人经验照亮一个普遍道理，但**不直接说出那个道理**。

### 写之前必须回答
1. **这篇文章的核心洞见是什么？** 一句话（不超过20字）
2. **我为什么有资格写这个？** 亲身经历/独特视角/深入研究
3. **读者读完会有什么感受或行动？** 如果答不出来，还没想清楚
4. **字数和平台？** Substack（1500-3000字最佳）/文学期刊（3000-8000字）/博客（800-1500字）

## 2. 结构工具箱

### 结构A：故事驱动型
最适合：个人经历类、回忆录式散文

```
钩子（100-200字）
  ↓ 一个具体的场景或时刻，制造悬念或情感张力
展开（正文主体）
  ↓ 故事+反思交替，每次反思不超过2-3句
  ↓ 中间有一个转折——理解发生了变化
收束（100-200字）
  ↓ 回到开头的意象，但带着新的理解
  ↓ 不说"所以"、"总之"、"因此"
```

### 结构B：论点驱动型
最适合：观点文章、文化评论、技术评论

```
钩子（100-200字）
  ↓ 反常识的断言 / 一个让人不安的问题 / 一个具体案例
核心论点（1-2句）
  ↓ 清晰、尖锐、可争论
论据1 + 案例/故事
论据2 + 案例/故事
论据3 + 案例/故事（可选）
  ↓ 每个论据用具体故事或数据支撑，不要纯说理
让步和反驳
  ↓ 诚实面对反对意见，然后解释为什么你的立场仍然成立
结尾
  ↓ 不是重复论点，而是把论点推进一步——"如果这是对的，那意味着……"
```

### 结构C：3-2-1 框架（Substack短文）
最适合：周更、轻量级洞见

```
3秒钩子：标题+第一句话解决核心问题
2分钟主体：一个具体故事展示你的观点
1个收获：读者带走的一件事
```

### 结构D：拼贴型
最适合：实验性散文、诗化散文

- 多个片段，不需要线性逻辑
- 片段之间通过意象、情绪、主题关联
- 整体效果大于各部分之和
- 风险最高——容易变成散乱的碎片

## 3. 钩子（开头）

开头的唯一任务：**让人想读下一段**。

### 六种有效的钩子

**1. 场景钩子**——直接进入一个画面
> 「三月的最后一个下午，我在医院走廊里第一次听到那个词。」

**2. 反常识钩子**——说一个违背直觉的事
> 「大多数关于效率的建议都是在教你如何更优雅地浪费时间。」

**3. 问题钩子**——提出一个读者也在想的问题
> 「为什么我们越忙越觉得空？」

**4. 数字钩子**——用数据制造冲击
> 「中国每年有4万篇小说投稿给文学期刊，发表率不到3%。」

**5. 对话钩子**——从一句话开始
> 「"你不适合写东西。"我妈在电话里说，语气像在说今天天气不错。」

**6. 故事钩子**——从一个微型故事开始
> 「去年冬天我删掉了一个写了三个月的长篇。不是因为写得不好，而是因为写得太好了——好到我怀疑它不是我写的。」

### 无效的开头（禁止使用）
- "在当今社会/时代……"
- "众所周知……"
- "所谓X，就是……"（定义式）
- "自古以来……"
- "随着XX的发展……"

## 4. Substack 特有策略

### 标题
- 标题是文章最重要的一句话——50%的读者根据标题决定是否点开
- 好标题的特征：具体、有张力、让人好奇
- 坏标题的特征：泛泛、正确但无趣、像论文题目
- 参考长度：8-15个字

### 副标题
- 补充标题没说的信息
- 降低读者的不确定性——"点进来我能得到什么"
- 不要重复标题

### 排版
- 段落保持短——手机阅读时长段落会变成文字墙
- 每3-4段考虑一个小标题或分隔线
- 引用和列表适度使用——打破纯文字的单调
- 配图：要么有信息量，要么不用——装饰性配图是噪音

### 发布节奏
- 固定频率比高频率重要——读者信任可预期的作者
- 质量波动不能太大——宁可少发不要凑数
- 免费文章和付费文章的比例：至少60%免费

### Sections 功能
- 用 Sections 区分不同类型的内容（散文/链接/问答/短想法）
- 让读者只订阅感兴趣的内容类型
- 不要把所有东西都堆在一个订阅流里

## 5. 散文的语言

### 与小说的区别
- 散文的"我"是真实的（或至少让读者相信是真实的）
- 散文允许直接思考——但节制比放纵好
- 散文的比喻可以更知识性——引用、典故、跨领域类比

### 散文特有的陷阱
- **自恋**——过度沉浸于自己的感受，忘记读者为什么要关心
- **鸡汤**——用情绪代替思考，用感动代替洞见
- **学术腔**——在非学术平台用论文语言
- **段子化**——为了有趣牺牲深度

### 引用的纪律
- 引用是为了借力，不是为了装饰
- 每次引用后必须用自己的话解释为什么引用这段
- 一篇文章不超过3-4处引用——多了就变成读书笔记

## 6. 改稿清单

每篇散文发表前过一遍：

1. **删掉前两段试试**——很多散文的真正开头在第三段
2. **把结尾的最后一句删掉**——看看是不是更好
3. **找到所有"我觉得"/"我认为"**——如果上下文已经清楚是你的观点，这些词多余
4. **每个论据都有具体案例吗？**——纯说理的段落读者会跳过
5. **大声朗读**——句子拗口的地方就是需要改的地方
6. **给一个不了解背景的人看标题和前三段**——他们想继续读吗？
