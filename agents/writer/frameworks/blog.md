# 博客文章写作框架（技术 + 非技术）

## 1. 博客的本质

博客文章 ≠ 散文 ≠ 论文。博客是**实用写作**：读者带着问题来，希望带着答案走。

### 写之前必须回答
1. **读者搜什么关键词会找到这篇？** 如果答不出来，你可能在写给自己
2. **读者的痛点是什么？** 具体到他们此刻卡在什么地方
3. **读者读完能做什么？** 一个具体的行动或决策
4. **目标字数？** 技术教程：1500-3000字 / 观点文：800-1500字 / 深度分析：3000-5000字

## 2. 博客文章类型和结构

### 类型A：教程型（How-to）
最适合：技术博客、教学内容

```
标题：如何[做到某件事]
引言（2-3句）：问题是什么 + 这篇文章能帮你解决
前置条件（可选）：读者需要什么背景知识/工具
步骤1：[标题] + 操作 + 代码/截图 + 解释
步骤2：...
步骤N：...
常见问题/陷阱（可选）
总结：3-5个要点回顾
```

**纪律：**
- 每一步必须可执行——读者应该能照着做
- 代码必须能跑——不要写伪代码当真代码
- 不要假设读者知道你知道的东西

### 类型B：观点型（Opinion）
最适合：行业评论、技术选型、文化评论

```
标题：[核心观点的浓缩]
钩子（2-3句）：一个引人注意的现象或问题
论点（1句）：你的核心主张
论据1 + 证据/案例
论据2 + 证据/案例
反对意见 + 你的回应
结尾：论点的延伸——"如果这是对的，那意味着……"
```

**纪律：**
- 观点必须可争论——如果没人会反对，你在说废话
- 每个论据用事实/数据/案例支撑——不要纯输出观点
- 诚实面对反对意见

### 类型C：清单型（Listicle）
最适合：资源汇总、工具推荐、经验总结

```
标题：[N]个[关于某主题的有用东西]
引言：为什么整理这个清单 + 筛选标准
条目1：标题 + 1-3句说明 + 为什么推荐
条目2：...
条目N：...
```

**纪律：**
- 数量要有理由——不要为了凑10个放3个凑数的
- 每个条目必须提供信息增量——不是换个说法重复
- 排序要有逻辑（重要性/难度/场景）

### 类型D：深度分析型（Deep Dive）
最适合：技术架构、案例分析、行业研究

```
标题：[某个复杂主题]的深度分析
摘要/TL;DR（3-5句）：核心结论
背景：为什么这件事值得深入分析
分析框架：你用什么角度来分析
分析主体：2-4个维度，每个维度有数据/案例
结论和启示
```

**纪律：**
- 必须有框架——不是想到什么写什么
- 数据和案例优于个人感受
- TL;DR 放在最前面——尊重忙碌的读者

## 3. 技术博客特有规范

### 代码
- 所有代码块标注语言（```python、```javascript等）
- 关键代码加注释——但不要每行都注释
- 代码能跑——发表前必须测试
- 版本号明确——"使用 Python 3.11+ 和 FastAPI 0.100+"

### 技术解释
- 先说"做什么"，再说"怎么做"，最后说"为什么"
- 类比优于抽象定义——用读者已经懂的东西解释新东西
- 图表优于大段文字——架构图、流程图、对比表
- 不要居高临下——"这很简单"、"显然"——对读者来说不一定简单

### 技术文章的致命错误
- 过时信息——发表后不维护
- 缺少前置条件——读者照着做跑不通
- 混淆自己的方案和最佳实践
- 把官方文档复述一遍——没有增量

## 4. 排版和可读性

### 标题
- 用数字、动词、具体名词
- 好标题："用 3 个 Git Hook 自动化你的代码质量检查"
- 坏标题："关于代码质量的一些想法"

### 段落
- 一段不超过4-5句
- 移动端阅读时长段落变成文字墙
- 每段开头的第一句话应该能独立成立——扫读者只看段首

### 视觉层级
- H2 作为主要章节标题
- H3 作为子章节
- 不要用 H4 以下——层级太深说明结构有问题
- 列表、引用块、代码块打破纯文字——但不要过度

### 强调
- **加粗**用于关键概念和重要结论（每段最多1-2处）
- *斜体*用于术语首次出现或引用
- 不要全段加粗——等于什么都没强调

## 5. SEO 基础（如果在乎流量）

- 标题包含目标关键词
- 前100字出现核心主题
- 用H2/H3组织内容——搜索引擎靠标题理解结构
- 有内部链接和外部链接
- 图片有 alt text
- URL 简短有意义

## 6. 发表前检查清单

1. **标题和前三句能让陌生人想读下去吗？**
2. **每个小节都提供了新信息吗？** 没有就合并或删
3. **代码能跑吗？**（技术文章）
4. **有没有未定义就使用的术语？**
5. **排版在手机上好看吗？** 预览移动端排版
6. **链接都能打开吗？**
7. **让一个不了解背景的朋友读——他能跟上吗？**
