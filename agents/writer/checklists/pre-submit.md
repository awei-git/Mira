# 发表/投稿前最终检查清单

文章定稿后、点发送前最后过一遍。

## 通用检查

### 内容
- [ ] 标题是否足够吸引人？（给3个不了解背景的人看标题，他们想点开吗？）
- [ ] 开头是否有效？（前100字能否留住读者？）
- [ ] 结尾是否有余味？（不是总结，是余音）
- [ ] 全文是否过了自我编辑四轮检查？（见 `self-edit.md`）
- [ ] 全文是否过了去AI味检查？（见 `anti-ai.md`）

### 格式
- [ ] 字数符合目标平台要求
- [ ] 标题/副标题/正文的字体字号正确
- [ ] 段落间距一致
- [ ] 中英文标点未混用
- [ ] 引用格式正确

### 元信息
- [ ] 作者署名正确
- [ ] 日期正确
- [ ] 分类/标签设置好
- [ ] 封面图/配图准备好（如需）

## Substack 特有

- [ ] 标题 ≤ 60字符（太长在手机上会截断）
- [ ] 副标题补充了标题没说的信息
- [ ] 预览文本（email预览）设置好
- [ ] Section 选择正确
- [ ] 免费/付费设置正确
- [ ] 预约发布时间设置好（如果不是立即发）
- [ ] 移动端预览过排版

## 博客/个人网站特有

- [ ] URL slug 简短有意义
- [ ] SEO 标题和描述填写（如果在乎搜索流量）
- [ ] 图片有 alt text
- [ ] 所有链接可点击
- [ ] 代码块语法高亮正确（技术文章）
- [ ] RSS 会正确推送吗？

## 文学期刊投稿特有

- [ ] 格式符合目标期刊要求（字体/字号/行距/页边距）
- [ ] 文件格式正确（.docx / .pdf）
- [ ] 作者信息块包含所有必要项（姓名/电话/邮箱/地址/字数/梗概）
- [ ] 投稿信/cover letter 准备好
- [ ] 是否需要匿名投稿（删除作者信息）？
- [ ] 记录投稿日期和平台到 `规格.md` 的投稿追踪表
- [ ] 确认未同时投给不接受一稿多投的平台

## 投稿后

- [ ] 记录投稿日期和状态到追踪表
- [ ] 设日历提醒：如果[N]天没回复则跟进
- [ ] 收到反馈后存入 `审稿意见_r[N].md`
