# 自我编辑检查清单

改稿时使用。每轮改稿只关注一个层面，按顺序执行。

## 第一轮：结构

- [ ] 删掉前两段，文章是否更好？（很多文章的真正开头在第三段）
- [ ] 每个段落/小节都提供了新信息吗？→ 没有就合并或删
- [ ] 段落/小节之间的顺序有逻辑吗？→ 试着交换位置，如果无所谓就说明逻辑不够
- [ ] 结尾是不是在重复开头说过的话？→ 结尾应该推进，不是总结
- [ ] 整体长度是否超出规格？→ 先砍段落再砍句子

### 小说/中篇额外检查
- [ ] 每章是否推进了情节/角色/主题？→ 空转章必须砍
- [ ] 章节之间的节奏是否有变化？→ 紧张-缓和交替
- [ ] 意象是否每次出现都推进了功能？→ 停在原地的意象多余

## 第二轮：逻辑

- [ ] 每个论点/情节转折有支撑吗？→ 没有证据/铺垫的断言不可信
- [ ] 有没有逻辑跳跃？→ 两个段落之间的因果链条是否完整
- [ ] 有没有自相矛盾的地方？
- [ ] 时间线一致吗？（小说）
- [ ] 人名/地名/术语一致吗？

## 第三轮：语言

- [ ] 大声朗读全文——句子拗口的地方就是需要改的地方
- [ ] 过 `checklists/anti-ai.md` 检查清单
- [ ] 搜索高频词，审视每一处是否必要
- [ ] 检查被动语态——能用主动语态的地方改过来
- [ ] 检查"的"的密度——三个"的"连在一起就要拆句子

## 第四轮：细节

- [ ] 错别字/语法错误
- [ ] 标点符号使用正确（中英文标点不混用）
- [ ] 数字和事实核查
- [ ] 链接是否有效（博客/Substack）
- [ ] 代码是否能运行（技术博客）
- [ ] 格式是否符合平台要求

## 最终检查

- [ ] 标题+前三句能让陌生人想读下去吗？
- [ ] 给一个不了解背景的人看——他能跟上吗？
- [ ] 你会把这篇文章转发给你尊敬的人看吗？→ 不会就还没到发表标准
