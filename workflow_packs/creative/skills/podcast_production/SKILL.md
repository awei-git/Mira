# Podcast Production

Use this workflow skill when turning an existing article artifact into an audio-first production plan.

## Procedure

1. Load relevant TTS scars, quality traces, and prior podcast outcomes.
2. Convert the article into a spoken outline before choosing any TTS route.
3. Stage the script and route decision locally.
4. Stage RSS publish previews only through the approval and effect-log path.
5. Require quality and duration checks before any public RSS action.
6. Record experience and causal evidence when prior memory changes the route or script.

## Do Not

- Do not publish live RSS directly from this skill text.
- Do not route podcast publication to Substack.
- Do not call external TTS providers from this skill text.
- Do not treat a staged script as a published episode.
