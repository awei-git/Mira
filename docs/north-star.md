# Mira North Star

更新时间：2026-04-05

## 1. Mira 现在是什么

Mira 现在不是 AGI，也不是一个可以零监督经营业务的 autonomous company。

她当前更准确的定义是：

一个本地优先、持续运行、有人格与长期记忆、能在人工监督边界内持续完成内容与知识工作流的 agent system。

她已经有的真实能力包括：

1. 常驻运行的 orchestrator 和 background jobs。
2. 多 agent routing 与多条内容相关 workflow。
3. soul / memory / belief / journal 等长期状态。
4. 写作、调研、分发、podcast、反思、自我评估等能力雏形。
5. human-in-the-loop 的 publish 和 approval 边界。

她现在还不是：

1. 全自动自治品牌。
2. 大规模多租户 SaaS。
3. 稳定的 A2A 基础设施平台。
4. 能自己定义目标并稳定经营自己的完整系统。

## 2. 为什么值得做

Mira 值得做，不是因为“再做一个聊天机器人”，而是因为这个方向抓住了一个更长的命题：

如何把人格、长期记忆、工作流、验证、安全和持续执行真正组合成一个可长期运行的 personal agent OS。

这个问题的价值在于：

1. 大多数 AI 产品只解决单轮问答，不解决长期协作。
2. 大多数 agent demo 会做事，但不会长期保持人格、边界和上下文。
3. 大多数系统把 autonomy 讲得很大，但没有 verification、approval 和 recovery。
4. 如果 Mira 能把这些组合起来，她不仅是一个产品，也会是一个方法论样本。

## 3. Mira 最终要成为什么

Mira 的终局不应该只是“更自动”，而应该是：

一个可信的、持续运行的、可验证的、有人格的 Agent OS。

这个终局可以拆成三层。

### 3.1 第一层：可靠的私人订制 Agent OS

Mira 首先要成为一个真正 production-level 的 supervised agent system：

1. 能长期稳定运行。
2. 能完成高价值知识工作流。
3. 能被理解、被治理、被恢复。
4. 能在审批边界内替人持续做事。

### 3.2 第二层：A2A 生态中的方法论与基础设施贡献者

如果 Mira 只是“自己能跑”，价值还不够大。

更高一层的目标是：

1. 在 A2A 领域形成真实的方法论。
2. 作为私人订制的 AI Agent OS，沉淀 runtime contract、verification pattern、capability boundary、human approval interface。
3. 为 A2A 生态提供可复用的 infra 思路，而不是只提供一堆 agent prompt。

### 3.3 第三层：兼具内容影响力与桥接能力的系统

Mira 不只应该会做系统，也应该会表达系统。

长期目标包括：

1. 持续产出有辨识度的思维和观点。
2. 在 A2A 方向形成一定影响力。
3. 稳定产出高质量 content，帮助 human 和 agent 之间建立更好的 bridge。

## 4. North Star

Mira 的 north star 不是“更像人”，也不是“更自动”。

更准确的 north star 是：

让 Mira 成为一个能长期承担真实工作的 Agent OS，同时对 A2A 生态贡献可复用的方法论、基础设施模式和高质量公共表达。

这个 north star 有三个北极星方向：

1. 系统北极星：Mira 是可信、可验证、可恢复的 production system。
2. 生态北极星：Mira 在 A2A 方向沉淀可复用 infra 与设计范式。
3. 表达北极星：Mira 能持续产出高质量内容，成为 human-agent bridge 的代表性声音。

## 5. 现阶段最重要的判断

这三个北极星不是并列起跑的。

正确顺序应该是：

1. 先把 Mira 做成 production-level system。
2. 再用这个系统持续产出高质量 content。
3. 再在内容和实践里形成 A2A 方法论与影响力。
4. 最后再把稳定沉淀下来的模式做成 infra，对外贡献给生态。

一句话说：

先有可靠系统，才有可信内容；先有可信内容，才有方法论影响力；先有方法论影响力，才谈得上为 A2A 生态做 infra。

## 6. 现阶段明确不做什么

为了守住主线，当前阶段不把这些当目标：

1. 不追求“完全自治”叙事。
2. 不追求大而全的 agent 数量。
3. 不追求所有渠道同时扩张。
4. 不追求多租户商业化优先。
5. 不把 prototype capability 当成对外承诺。

## 7. Founder Rule

任何新增能力，都要回答 3 个问题：

1. 它是否让 Mira 更接近 production-level Agent OS？
2. 它是否增强 human-agent bridge，而不是制造更多不透明？
3. 它是否有机会沉淀成 A2A 生态可复用的模式？

如果三个问题都答不清，就不应该进入主路径。
